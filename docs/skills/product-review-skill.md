# Product Review Skill

## Purpose

Review KnowNext.ai changes from a product engineering perspective.

## Context

Product quality depends on technical foundation and a coherent desktop documentation workflow.

## Rules

- Review against the product goal first.
- Check that stack constraints are honored.
- Verify UI constraints from screenshots.
- Confirm no forbidden Git branch UI appears.
- Confirm Milkdown remains the editor.
- Confirm backend contracts remain organized.

## Recommended Steps

1. Read the changed files by feature.
2. Compare behavior with acceptance criteria.
3. Check architectural boundaries.
4. Run available tests.
5. Update docs or checklist for gaps.

## Acceptance Criteria

- Findings are concrete and tied to files.
- Product risks are separated from implementation details.
- Follow-up work is explicit.

## Mistakes To Avoid

- Treating visual drift as subjective when constraints are explicit.
- Accepting raw Markdown as the main experience.
- Ignoring documentation and agent guidance changes.
