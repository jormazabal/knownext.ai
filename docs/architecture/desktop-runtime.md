# Desktop Runtime

KnowNext.ai uses Tauri as the desktop shell.

## Current Runtime

- Tauri loads the Vite React app.
- Tauri registers the updater and process plugins for runtime self-update checks.
- FastAPI is run manually during development.
- The frontend can use local mocks or call FastAPI with `VITE_USE_BACKEND=true`.

## Auto Updates

The desktop app checks public GitHub Releases through the Tauri updater plugin.

- Endpoint: `https://github.com/jormazabal/knownext.ai/releases/latest/download/latest.json`
- Update artifacts are generated by `.github/workflows/release.yml`.
- Windows uses passive install mode and prefers the NSIS bundle in `latest.json`.
- The frontend accesses updater operations only through `apps/desktop/src/lib/runtime/updater.ts`.
- The account menu exposes a manual `Buscar actualizaciones` action.
- The app performs a silent automatic check shortly after startup.
- Installation is always user-confirmed and attempts to persist pending drafts before downloading the update.

Updater signatures are mandatory. The public key lives in `apps/desktop/src-tauri/tauri.conf.json`; the private key and password live only in local secret storage and GitHub Actions secrets.

## Future Sidecar Plan

Tauri should start FastAPI as a sidecar process when the desktop app opens.

The sidecar plan must include:

- Allocate or reserve a local port.
- Start FastAPI with explicit host `127.0.0.1`.
- Pass the selected port to the frontend.
- Health-check the backend before enabling backend-backed actions.
- Shut down FastAPI when the desktop app closes.
- Avoid exposing the API beyond localhost.

## Security Notes

- Keep CORS restricted to Tauri and local development origins.
- Do not allow arbitrary shell access from React.
- Route filesystem, Git, and AI operations through backend services.
- Never expose API keys in the frontend.
- Never expose the updater private signing key or password in the repository or frontend.
