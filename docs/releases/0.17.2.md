# KnowNext.ai 0.17.2

## Summary

KnowNext.ai 0.17.2 fixes document-tree folder expansion so the tree no longer reopens the first levels after backend refreshes, and preserves the user's folder layout per project.

## Fixes

- Starts newly loaded project trees with folders collapsed by default.
- Persists expanded folder paths per project in `config.json`.
- Restores the saved folder state after reopening the application.
- Keeps user-changed folder state stable when the tree is refreshed after file operations, imports, sync, AI operations, or orphan-draft restore.

## Implementation Notes

- Backend filesystem scanning now emits folder nodes as closed by default.
- Frontend tree state is stored by normalized folder path rather than transient node ids, so it survives backend tree reconstruction.
- The app reapplies current or persisted folder state whenever a refreshed tree arrives from the API.
- Config validation normalizes persisted tree paths and rejects invalid traversal-style entries.

## Validation

- `pnpm --filter @knownext/desktop test`
- `python -m pytest backend/tests`
- `pnpm --filter @knownext/desktop build`
- `node scripts/check-version.mjs`
- Browser validation on `http://127.0.0.1:1424/` with a nested project tree, including default collapsed state, persisted expand/collapse after reload, and stable state after creating a folder.

## Distribution

The GitHub release for `v0.17.2` must include the NSIS installer, MSI installer, both updater signatures, and `latest.json`. The published `latest.json` must resolve to version `0.17.2` and keep `windows-x86_64.url` pointing to the MSI artifact.
