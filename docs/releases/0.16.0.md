# KnowNext.ai 0.16.0

Release date: 2026-05-17

## Summary

KnowNext.ai 0.16.0 adds a protected external-change workflow for local Git projects. When a user pastes files or folders into a project from Windows Explorer, the app detects the working-tree changes, classifies risk, and helps import safe documentation into the project history without exposing low-level Git concepts in the UI.

## Product Changes

- Added a global sync indicator in the document tab row with human states such as `Revisión necesaria`, `Guardando versión`, `Pendiente de sincronizar`, and `Sincronizado`.
- Added a contextual banner above the active document when external changes are detected.
- Added a right-side review drawer that groups folders, Markdown documents, images, attachments, private files, ignored files, and unsupported files.
- Added temporary `Nuevo`, `Modificado`, `Eliminado`, and `Revisar` badges in the document tree.
- Added `Importar seguros`, `Importar seleccionados`, and `Omitir todo` actions for the external-change queue.
- Kept private or technical files such as `.env`, keys, certificates, `.git`, `node_modules`, `dist`, and `build` out of automatic import.

## Technical Changes

- Added FastAPI contracts for `ExternalChangeSet`, external-change items, import decisions, and import results.
- Added `external_changes_service` to classify local Git working-tree changes and create a backend-owned local version for selected paths.
- Extended `git_service` with porcelain status, project-level version creation, and remote-origin checks.
- Added frontend API access under `apps/desktop/src/lib/api/externalChanges.ts`.
- Added feature components for the banner, drawer, and sync indicator under `apps/desktop/src/features/externalChanges`.
- Added backend coverage for scanning, classifying, importing safe changes, and omitting private files from versioning.

## Known Limitations

- Automatic external-change detection currently requires a local Git working tree.
- GitHub API cache projects return an unsupported-detection state until a cache baseline strategy is defined.
- Large attachments and deletions are surfaced for review but are not merged automatically.

## Validation

- `pnpm --filter @knownext/desktop build`
- `pnpm --filter @knownext/desktop test -- --run`
- `python -m pytest backend\tests\test_api.py -q`
- Browser validation against a temporary FastAPI-backed local Git project with pasted Markdown files and a blocked `.env`.
