# KnowNext.ai 0.7.1

KnowNext.ai 0.7.1 hardens the local runtime boundary so the web app, installed Windows app, and different application versions can run without sharing the wrong backend process.

## Changes

- Added a stronger `/health` contract with app identity, schema version, runtime profile, endpoint, active port, manager, instance id, startup time, version, and app-data directory.
- Separated browser development and packaged desktop profiles so the frontend rejects incompatible local APIs instead of timing out against the wrong backend.
- Added automatic backend port selection for packaged desktop builds. The app prefers `8765`, scans through `8799`, and skips occupied or incompatible services.
- Added fixed-port configuration in the Services settings panel, with apply-and-restart behavior and rollback if the new runtime cannot start.
- Added clearer Services diagnostics for active profile, expected profile, endpoint, port, managed-by value, instance, app-data path, sidecar path, last error, and copyable diagnostic detail.
- Fixed AI document edit handling so prompts like `Redacta una receta de cocina` update the active Milkdown editor buffer as unsaved content.
- Hardened API readiness, AI request timeouts, profile mismatch messaging, and backend recovery behavior.
- Updated runtime, backend, development, and manual QA documentation for the new port/profile model.

## Verification

- Run `pnpm release:check`.
- Confirm the installed app starts its packaged backend with `profile=desktop`.
- Confirm the browser development app starts against a backend with `profile=web-dev`.
- Occupy port `8765` with an unrelated process and confirm the packaged app starts on the next available automatic port.
- Configure a fixed occupied port and confirm the app reports the failure without persisting a broken runtime state.
- Open `Ajustes > Servicios` and confirm endpoint, profile, port, app-data path, restart availability, and diagnostics are coherent.
- Submit `Redacta una receta de cocina` against an active document and confirm the editor receives the generated recipe as unsaved content.
- Create and delete a folder/document in a browser profile and confirm the tree and backend stay synchronized.
- Confirm `latest.json` resolves to `0.7.1`.
- Confirm `windows-x86_64.url` in `latest.json` points to `KnowNext.ai_0.7.1_x64_en-US.msi`.
- Confirm the README installer URL resolves to `KnowNext.ai_0.7.1_x64-setup.exe`.
