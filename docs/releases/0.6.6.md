# KnowNext.ai 0.6.6

KnowNext.ai 0.6.6 publishes the packaged backend sidecar and readable logging release with the backend version metadata included in the executable.

## Highlights

- The Windows app bundles `knownext-backend` as a Tauri sidecar and starts it in release builds.
- Tauri waits for the local API `/health` endpoint on `127.0.0.1:8765` before normal frontend API calls.
- The backend uses the Tauri app data directory through `KNOWNEXT_APP_DATA_DIR`.
- Trace entries are written as readable timestamped blocks with `Message` and optional `Detail` sections.
- The sidecar build includes all backend `app.*` modules and the root `VERSION` file so `/health` reports the packaged version.
- The release workflow validates the sidecar `/health` endpoint before creating installer artifacts.
