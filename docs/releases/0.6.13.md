# KnowNext.ai 0.6.13

KnowNext.ai 0.6.13 improves backend resilience and gives users direct visibility into local service health from application settings.

## Changes

- The application settings dialog now includes a Services section focused on the local backend.
- The Services section shows backend status, endpoint, active and expected versions, app data profile, sidecar executable path, last detected error, and last check time.
- Users can manually refresh service status and request a backend restart from settings.
- Installed desktop builds now supervise the bundled backend and automatically attempt a restart when `/health` stops matching the expected version and app data profile.
- Backend startup, shutdown, supervisor recovery, manual restarts, and health-check failures are recorded in `knownext.log` with actionable detail.
- Frontend tests cover the new Services settings panel.
