# KnowNext.ai 0.6.7

KnowNext.ai 0.6.7 fixes packaged backend startup and changes the Windows updater path to avoid unnecessary administrator prompts for normal per-user installs.

## Changes

- The desktop runtime now resolves the packaged backend executable from the installed app directory, including the NSIS layout where `knownext-backend.exe` is installed next to `knownext-ai-desktop.exe`.
- Backend startup logs now include the executable path used by the desktop app and capture backend stdout/stderr in `knownext.log`.
- The release workflow now generates `latest.json` with the NSIS setup artifact as the Windows updater URL.
- MSI remains published as a release asset for environments that explicitly need it, but it is no longer the default in-app update path.

## Verification

- Confirm `latest.json` resolves to `0.6.7`.
- Confirm `windows-x86_64.url` in `latest.json` points to `KnowNext.ai_0.6.7_x64-setup.exe`.
- Confirm a packaged Windows install starts the local FastAPI backend and `http://127.0.0.1:8765/health` returns `status=ok` and `version=0.6.7`.
- Confirm updating a per-user install does not request administrator permissions.
