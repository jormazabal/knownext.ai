# KnowNext.ai 0.6.10

KnowNext.ai 0.6.10 fixes project creation when a new local folder is prepared for GitHub synchronization.

## Changes

- Empty local Git repositories are now treated as projects with no versions yet when the workspace asks for versioning status.
- The "Create GitHub repo from local" flow no longer shows `fatal: Needed a single revision` immediately after project creation.
- Added backend coverage for querying versioning status right after creating a new local folder and GitHub repository configuration.
