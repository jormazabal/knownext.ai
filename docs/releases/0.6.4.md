# KnowNext.ai 0.6.4

KnowNext.ai 0.6.4 packages and starts the local FastAPI backend from the Tauri runtime, and makes diagnostic logs easier to read.

## Highlights

- The Windows app now bundles `knownext-backend` as a Tauri sidecar and starts it in release builds.
- Tauri waits for the local API `/health` endpoint on `127.0.0.1:8765` before normal frontend API calls.
- The backend receives the Tauri app data directory through `KNOWNEXT_APP_DATA_DIR`, keeping config, drafts, and logs in the same app profile.
- Trace entries are written as readable timestamped blocks with `Message` and optional `Detail` sections instead of compact JSON lines.
- Suppressed background API connection failures are no longer written repeatedly while the sidecar is starting or unavailable.
- The release workflow builds the backend sidecar before producing NSIS, MSI, updater signatures, and `latest.json`.
