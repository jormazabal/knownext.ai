# KnowNext.ai 0.17.1

## Summary

KnowNext.ai 0.17.1 refines support-file handling in the document tree and makes generic project files manageable alongside Markdown documentation.

## Features

- Adds distinct visual identification for PDF, DOCX, PPTX, XLSX, TXT, and CSV files in the tree and search results.
- Uses a generic file icon for every other non-private support file type.
- Keeps the `Archivos` tree toolbar fixed so search, add, view, and settings actions stay visible while the folder/file list scrolls.
- Treats unknown non-private files as support-file attachments for scan, import, rename, move, delete, backup, and Git/version flows.

## Implementation Notes

- The tree toolbar now sits outside the scroll container; only the tree list owns vertical overflow.
- Frontend import delegates unknown non-Markdown and non-image files to the support-file attachment flow.
- Filesystem and external-change services no longer restrict attachments to a fixed extension allowlist, while still excluding private/secrets files.
- Known readable file types remain the only attachment types exposed as AI context.

## Validation

- `pnpm --filter @knownext/desktop test -- DocumentTree`
- `python -m pytest backend/tests/test_api.py -k "attachment_files or external_changes_scan"`
- `pnpm --filter @knownext/desktop build`
- Browser validation with PDF, DOCX, PPTX, XLSX, TXT, CSV, and generic `.bin` files, including fixed tree-toolbar scroll behavior.

## Distribution

The GitHub release for `v0.17.1` must include the NSIS installer, MSI installer, both updater signatures, and `latest.json`. The published `latest.json` must resolve to version `0.17.1` and keep `windows-x86_64.url` pointing to the MSI artifact.
