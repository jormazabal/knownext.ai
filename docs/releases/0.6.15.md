# KnowNext.ai 0.6.15

KnowNext.ai 0.6.15 focuses on the browser/debug experience, managed project storage, project editor UX, and RAG-related startup noise.

## Changes

- Browser-created projects now use managed backend storage under the web profile instead of asking the browser for unrestricted local folder paths.
- The project creation assistant adapts to browser mode, shows managed storage clearly, and avoids folder picker flows that cannot work reliably from a normal browser.
- The project editor is more compact and separates editable identity fields from read-only project behavior. In browser mode it no longer exposes backend filesystem paths.
- Initial API loading now waits for local API readiness and retries transient GET failures, reducing false connection errors during refresh/startup.
- RAG index status is only requested when indexing is enabled. The backend index-status response is also fixed for enabled RAG flows.
- JSON project persistence is more robust on Windows when multiple operations write close together.
