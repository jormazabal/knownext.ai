# KnowNext.ai 0.15.0

Release date: 2026-05-16

## Summary

KnowNext.ai 0.15.0 adds a compact, keyboard-first search modal for finding folders and Markdown documents by name from the project navigation tree. The feature focuses strictly on names, not document content, and keeps results understandable by showing the containing folder path for each result.

## Product Changes

- Added a search modal opened from the existing search icon above the document tree.
- Kept the sidebar compact by avoiding a permanent search input in the tree header.
- Added incremental suggestions for folders and Markdown documents whose names contain the typed text anywhere in the name.
- Added parent-folder context below each result, with long paths shortened from the left.
- Added inline completion for the first clear prefix match in the input.
- Added keyboard handling for Escape, Tab, Enter, ArrowUp, and ArrowDown.
- Added selection behavior that opens documents in the editor and expands parent folders in the tree.
- Added folder selection behavior that highlights and expands the selected folder without replacing the current document.
- Refined result highlighting so matched letters do not visually split words.

## Technical Changes

- Added a typed document-name search utility with ranking, normalized matching, match ranges, and path metadata.
- Added tree-selection state separate from the active document so folders can be visibly selected.
- Added parent expansion and scroll-into-view behavior after search selection.
- Excluded image assets and document content from the search surface.
- Added component tests for fragment search, folder selection, document opening, and keyboard acceptance.
- Updated manual validation coverage for the name-only search flow.

## Validation

- `pnpm --filter @knownext/desktop test -- DocumentTree documentNameSearch`
- `pnpm --filter @knownext/desktop build`
- Browser validation against a real FastAPI-backed project with nested folders and documents.
