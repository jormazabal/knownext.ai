# KnowNext.ai 0.6.11

KnowNext.ai 0.6.11 fixes update startup, data-profile recovery, and diagnostics after the 0.6.10 rollout.

## Changes

- The desktop shell now verifies that an already-running local API matches the bundled app version and the expected app data directory before reusing it.
- If an incompatible local API is still listening on the backend port after an update, the app records the mismatch and starts the bundled backend instead.
- The backend now recovers projects, configuration, and credentials from the previous KnowNext.ai data profile when the current profile only contains empty defaults.
- Controlled API errors and unhandled backend exceptions are written to `knownext.log` with endpoint and detail information even when detailed trace logging is disabled.
- Frontend startup now preserves loaded projects if a secondary startup request such as the active project's versioning status fails.
