# KnowNext.ai 0.6.14

KnowNext.ai 0.6.14 fixes the remaining backend/profile issues seen after 0.6.13, especially when using the app from both the Windows installation and a local browser build.

## Changes

- Desktop and browser builds now use separate local API ports and data profiles by default: `8765` for the installed/Tauri app and `8766` for browser development.
- Local browser ports are accepted by backend CORS, so Vite pages such as `127.0.0.1:1421` can call the API without preflight failures.
- Seeded mock projects (`Proyecto Alpha`, `Proyecto Beta`, `Proyecto Gamma`) are no longer exposed at runtime. If a valid recovered `projects.json.corrupt-*` backup exists, it is restored before falling back to an empty registry.
- JSON persistence now uses per-file locks and unique temporary files to prevent Windows file replacement conflicts during concurrent requests.
- Browser backend startup is documented through `pnpm backend:web`, using a dedicated web profile so app configuration and project configuration can differ from the installed app.
