# KnowNext.ai 0.17.3

## Summary

KnowNext.ai 0.17.3 tightens document navigation from open tabs and fixes Markdown list rendering inside the visual editor.

## Fixes

- Reveals the active document in the navigation tree whenever the user activates a document tab.
- Expands every parent folder needed to make the active document visible.
- Scrolls the tree to the selected document row when the selected node is outside the visible area.
- Returns the tree filter to the full view when the active document is hidden by a file-type filter.
- Aligns numbered and bulleted lists in the Milkdown editor without duplicate browser and Milkdown markers.

## Implementation Notes

- Centralized document-tree reveal behavior in the application shell so document opens, tab activation, and tab close fallback keep tree selection consistent.
- Memoized filtered tree nodes and added containment checks so the tree can recover from filters that would hide the active selection.
- Scoped list styling separately for normal HTML list items and Milkdown `li.list-item` nodes, preserving Milkdown's custom labels while disabling native duplicate markers.

## Validation

- `pnpm --filter @knownext/desktop test -- DocumentTree treeOpenState`
- `pnpm --filter @knownext/desktop build`
- Browser validation on `http://127.0.0.1:1424/` with nested folders and two open document tabs, including tab activation after collapsing the parent folder and visual inspection of numbered and bulleted lists.

## Distribution

The GitHub release for `v0.17.3` must include the NSIS installer, MSI installer, both updater signatures, and `latest.json`. The published `latest.json` must resolve to version `0.17.3` and keep `windows-x86_64.url` pointing to the MSI artifact.
