# KnowNext.ai 0.4.2

KnowNext.ai 0.4.2 prevents the installed desktop app from showing a backend connection error on first launch and prepares optional Windows Authenticode signing for future releases.

## Changes

- Added an optional Authenticode signing hook for Windows Tauri bundles.
- Added GitHub Actions certificate import for signed Windows release artifacts when a certificate is configured.
- Changed packaged desktop builds to use local mock services by default until FastAPI is packaged as a managed sidecar.
- Kept local development using FastAPI by default.

## Windows Installer Trust

The open source release may ship without Windows Authenticode signing because publicly trusted code signing certificates are expensive for a small open source project. Windows SmartScreen or endpoint protection may warn before running the installer.

The Tauri updater signature is still mandatory and remains separate from Authenticode.

For manual verification, compare the installer SHA256 shown on the GitHub release asset with the local file hash:

```powershell
Get-FileHash .\KnowNext.ai_0.4.2_x64-setup.exe -Algorithm SHA256
```

If a Windows code signing certificate becomes available, configure these GitHub Actions secrets:

- `WINDOWS_CERTIFICATE`: base64-encoded `.pfx` code signing certificate.
- `WINDOWS_CERTIFICATE_PASSWORD`: export password for the `.pfx` certificate.

The updater signing secrets remain separate:

- `TAURI_SIGNING_PRIVATE_KEY`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

## Validation

- Run `pnpm release:check`.
- Confirm CI signs artifacts when `WINDOWS_CERTIFICATE` is configured, or clearly documents that Authenticode was skipped.
- Confirm `latest.json` resolves to version `0.4.2`.
