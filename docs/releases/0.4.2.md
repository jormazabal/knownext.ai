# KnowNext.ai 0.4.2

KnowNext.ai 0.4.2 prepares signed Windows releases and prevents the installed desktop app from showing a backend connection error on first launch.

## Changes

- Added an Authenticode signing hook for Windows Tauri bundles.
- Added GitHub Actions certificate import for signed Windows release artifacts.
- Changed packaged desktop builds to use local mock services by default until FastAPI is packaged as a managed sidecar.
- Kept local development using FastAPI by default.

## Required Secrets

Signed Windows releases require these GitHub Actions secrets:

- `WINDOWS_CERTIFICATE`: base64-encoded `.pfx` code signing certificate.
- `WINDOWS_CERTIFICATE_PASSWORD`: export password for the `.pfx` certificate.

The updater signing secrets remain separate:

- `TAURI_SIGNING_PRIVATE_KEY`
- `TAURI_SIGNING_PRIVATE_KEY_PASSWORD`

## Validation

- Run `pnpm release:check`.
- Confirm CI signs `knownext-ai-desktop.exe`, the NSIS installer, and MSI artifacts with a valid Authenticode signature.
- Confirm `latest.json` resolves to version `0.4.2`.
