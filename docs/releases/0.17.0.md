# KnowNext.ai 0.17.0

## Summary

KnowNext.ai 0.17.0 adds first-class support-file management for files that live beside Markdown documentation but are not Markdown documents or project images.

## Features

- Adds support files to the project tree for formats such as PDF, DOCX, XLSX, PPTX, TXT, CSV, TSV, JSON, YAML, XML, ZIP, 7z, and RAR.
- Shows support files with file-type icons and includes them in tree search, selection, filtering with `Solo archivos`, rename, move, delete, backup, and Git/version flows.
- Keeps support files out of document/image viewers so selecting one preserves the current workspace tab while making the file active in the tree.
- Adds unified `Importar archivo` handling for Markdown, images, and support files.
- Exposes readable support files as explicit AI prompt context through backend-owned extraction, while unsupported binary/archive files remain manageable project files without silent empty AI context.

## Implementation Notes

- `filesystem_service` now emits `attachment` tree nodes, filters private dot/secrets files, validates supported suffixes, and imports support files through a dedicated endpoint.
- `ai_context_service` can resolve project support files from `@` lookup or tree actions and store extracted text as temporary prompt context.
- External-change detection now counts support files separately and keeps them in the review path.
- React tree/search/context UI treats support files as a separate type with file-specific actions and no viewer behavior.

## Validation

- `pnpm version:check`
- `pnpm release:check`
- Backend API coverage for support-file tree scanning, import, rename, hidden/private omission, external-change summaries, and project AI context extraction.
- Frontend coverage for support-file tree rendering, filtering, search, context menu actions, and no-editor activation.
- Browser validation with Markdown, image, CSV, TXT, PDF, and hidden `.env` files in a real project.

## Distribution

The GitHub release for `v0.17.0` must include the NSIS installer, MSI installer, both updater signatures, and `latest.json`. The published `latest.json` must resolve to version `0.17.0` and keep `windows-x86_64.url` pointing to the MSI artifact.
