# KnowNext.ai 0.10.0

KnowNext.ai 0.10.0 adds explicit AI context management directly inside the prompt so users can see, adjust, and trust which project and external sources will be sent with each AI request.

## Changes

- Added `@` references in the AI prompt for active project documents, with incremental filtering and inline selection.
- Added persistent prompt chips for project documents, external files, and images; visible chips are the sources included in subsequent AI requests until removed or expired.
- Added external context ingestion for Markdown, plain text, PDF, DOCX, PPTX, and image files through upload, drag-and-drop, and paste flows.
- Added a sources popover with source type, status, expiry, preview, manual removal, one-hour extension, and add-to-project conversion for external text sources.
- Added expiry-aware context handling so stale prompt sources are removed automatically while near-expiry sources show a warning state.
- Added backend context storage, extraction, preview, project conversion, source resolution, and conversation source tracing through stable FastAPI contracts.
- Added OpenAI Responses payload support for explicit text sources and image inputs while keeping React provider calls mediated by FastAPI.
- Refined the prompt context chip visual state to match the KnowNext.ai AI surface: compact grey chips with pale orange hover, and warning tooltips only for expiring, warning, or error states.

## Verification

- Run `pnpm release:check`.
- In the AI prompt, type `@` and confirm project documents are listed and filtered while typing.
- Select a project document reference and confirm it appears as a visible chip inside the prompt before sending.
- Drag or upload a Markdown, PDF, DOCX, PPTX, or image file and confirm it appears as an active source chip.
- Open the sources popover and confirm status, source type, expiry, preview, removal, and one-hour extension controls work.
- For an external text source, use `Añadir al proyecto` and confirm a Markdown document is created in the active project.
- Send a prompt with active sources and confirm the user message records the sources used in the conversation.
- Confirm normal source chips use grey text/icons and only pale orange background/border on hover.
- Confirm `latest.json` resolves to `0.10.0`.
- Confirm `windows-x86_64.url` in `latest.json` points to `KnowNext.ai_0.10.0_x64_en-US.msi`.
