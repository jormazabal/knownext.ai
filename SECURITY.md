# Security Policy

## Supported Versions

The latest published release receives security fixes.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability.

Use GitHub Security Advisories / private vulnerability reporting for this
repository when available. If that is not available, contact the repository
maintainer privately through GitHub.

Include:

- Affected version or commit.
- Steps to reproduce.
- Impact and affected data, if known.
- Any suggested mitigation.

Security reports are handled in good faith. Public disclosure should wait until
a fix or mitigation is available.
